# Maze Generator & Solver

CLI-приложение для генерации лабиринтов и поиска кратчайшего пути в них. Поддерживает несколько алгоритмов генерации и решения, вывод в консоль или файл, ASCII и Unicode-рендеринг.

## Стек технологий

| Компонент | Технология |
|---|---|
| Язык | Go 1.25 |
| CLI-фреймворк | spf13/cobra v1.10 |
| Линтер | golangci-lint |
| Тесты | стандартный testing |

## Быстрый старт

```bash
make build                  # собрать бинарник ./bin/app
```

```bash
# Сгенерировать лабиринт 21x21 алгоритмом DFS и сохранить в файл
./bin/app generate --algorithm=dfs --width=21 --height=21 --output=maze.txt

# Найти путь алгоритмом A*
./bin/app solve --algorithm=astar --input=maze.txt --start=0,0 --finish=20,20
```

## Команды

```
generate    Сгенерировать лабиринт
solve       Найти путь в лабиринте
```

### generate

| Флаг | По умолчанию | Описание |
|---|---|---|
| `--algorithm` | `dfs` | Алгоритм генерации: `dfs`, `prim`, `randwalk` |
| `--width` | — | Ширина лабиринта (нечётное число) |
| `--height` | — | Высота лабиринта (нечётное число) |
| `--output` | — | Путь к выходному файлу (если не задан — вывод в консоль) |
| `--unicode` | `false` | Использовать Unicode box-drawing символы |

### solve

| Флаг | По умолчанию | Описание |
|---|---|---|
| `--algorithm` | `astar` | Алгоритм поиска пути: `astar`, `dijkstra`, `spfa` |
| `--input` | — | Путь к файлу с лабиринтом |
| `--start` | — | Начальная точка в формате `x,y` |
| `--finish` | — | Конечная точка в формате `x,y` |
| `--output` | — | Путь к выходному файлу (если не задан — вывод в консоль) |

## Архитектура

Проект следует Clean Architecture: domain → application → infrastructure → cmd.

```
cmd/app/
  main.go           — точка входа
  cmd/              — cobra-команды (generate, solve)
internal/
  domain/
    cellType/       — тип клетки (стена / проход)
    point/          — 2D-координата
    maze/           — структура лабиринта
    path/           — маршрут от старта до финиша
    generator/      — интерфейс и реализации генераторов
    solver/         — интерфейс и реализации решателей
    IO/             — интерфейсы рендерера и записи
  application/
    generating/     — use case генерации
    solving/        — use case решения
  infrastructure/
    printChar/      — ASCII-рендерер
    printUnicode/   — Unicode-рендерер
    readFile/       — чтение лабиринта из файла
    writeConsole/   — запись в stdout
    writeFile/      — запись в файл
test/
  cases/            — интеграционные тест-кейсы
```

## Алгоритмы

### Генерация

| Алгоритм | Флаг | Описание |
|---|---|---|
| DFS | `dfs` | Рекурсивный поиск в глубину с бэктрекингом |
| Prim | `prim` | Модифицированный алгоритм Прима (случайный выбор рёбер) |
| Random Walk | `randwalk` | Случайное блуждание (алгоритм Олдауса–Бродера) |

### Поиск пути

| Алгоритм | Флаг | Описание |
|---|---|---|
| A* | `astar` | Эвристический поиск с манхэттенским расстоянием |
| Dijkstra | `dijkstra` | Алгоритм Дейкстры с min-heap |
| SPFA | `spfa` | Shortest Path Faster Algorithm на основе очереди |

## Принятые решения

**1. Нечётные размеры лабиринта**
Генераторы работают со стеновой решёткой: чётные индексы — стены, нечётные — клетки. Поэтому ширина и высота должны быть нечётными. Приложение отклоняет чётные значения с поясняющей ошибкой.

**2. Два формата рендеринга**
ASCII-рендерер (`#`, `.`) даёт максимальную совместимость с терминалами и файлами. Unicode-рендерер использует box-drawing символы для наглядного отображения — включается флагом `--unicode`.

**3. Разделение IO через интерфейсы**
`MazePrinter` и `MazeWriter` инжектируются в use case через интерфейсы из `domain/IO`. Это позволяет тестировать бизнес-логику без реальной файловой системы и подменять рендерер без изменения ядра.
